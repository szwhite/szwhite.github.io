# 基本概念

Spring Cloud Gateway

Spring Cloud Gateway是基于Spring生态系统之上构建的 API 网关 ，包括：Spring 5.x ，Spring Boot 2.x 和 Project Reactor。

Spring Cloud Gateway提供一种简单而有效的方法来控制路由到API ，并为它们提供跨领域的关注点，例如：安全性，监视/指标，限流等。

什么是网关服务

API Gateway（APIGW / API 网关）， 是系统对外的唯一入口。 Api 网关封装了系统内部架构，为每个客户端提供定制的API。

为什么要使用网关

微服务应用部署在不同机房，不同服务器等情况下，客户端想要请求对应的服务就需要知道具体的IP或者域名URL，当微服务数量过多时是相当麻烦的，所以就有了网关，客户端直接发送请求到网关，由网关来根据请求标识解析出微服务所在的位置，再把请求转发到对应的微服务实例，

核心概念

路由（Route）：路由是网关最基础的部分，组成有：ID，目标URL，一组断言和一组过滤器组成。如果断言路由为true，则说明请求的url和配置匹配

断言（Predicate）：java8中的断言函数，用来定义请求地址，参数信息 如：- Path = /请求地址/** (表示该请求地址下的所有请求) - Method=GET,POST 等

过滤器（Filter）：Spring Cloud Gateway中的Filter分为两种类型（Gateway Filter，Global Filter）。过滤器会对请求和响应进行处理。

# 使用网关

1.添加依赖

<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-gateway</artifactId>
</dependency>

2.添加配置文件 resources/application.yml

server:
  port: 8500
spring:
  application:
    name: gateway-server #服务名
  cloud:
      gateway:
        routes:
         - id: realtime
           uri: lb://USER-SERVER #服务名
           order: 0
           predicates:
            - Path=/api/cluster_node/**
           filters:
            - StripPrefix=2 #默认为true 也可设置为1 也就是将Path去除前缀 /api/cluster_node/** = /cluster_node/** 设置为2不去除

# 路由规则

Spring Cloud Gateway 创建 Route 对象时，使用 RoutePredicateFactory 创建 Predicate 对象，Predicate 可以赋值给Route

  • Spring Cloud Gateway可以包含多个内置的 Route Predicate Factories。
  • 所有断言都匹配HTTP请求的不同属性
  • 多个 Route Predicate Factories 可以通过 （and） 结合在一起使用
  • 路由断言工厂 RoutePredicateFactory 包含的主要实现类包括 DateTime，请求的远端地址，路由权重，请求头，Host地址，请求方法，请求路径和请求参数等类型的路由断言。
server:
  port: 8500
spring:
  application:
    name: gateway-server #服务名
  cloud:
      gateway:
        routes:
         - id: realtime
           uri: lb://USER-SERVER #服务名
           order: 0
           predicates:
            - Path=/api/cluster_node/** #请求路径
            - After=2021-02-23T14:20:00.000+08:00[Asia/Shanghai] # DateTime
            - Method=GET,POST #请求方法
           filters:
            - StripPrefix=2

# 路由配置

在Spring Cloud Gateway中配置URL有三种方式

  1. websocket 配置方式

    server:
  port: 8500
spring:
  application:
    name: gateway-server #服务名
  cloud:
      gateway:
        routes:
         - id: realtime
           uri: ws://localhost:9000/ # websocket 配置方式
           order: 0
           predicates:
            - Path=/api/cluster_node/** #请求路径
