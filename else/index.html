# OSS以及配置Typora 图片上传

目的：实现使用Typora写文档时 插入图片自动上传到阿里云OSS对象存储服务 实现网络访问

OSS文档链接：https://help.aliyun.com/document_detail/31883.html?spm=a2c6h.13066369.0.0.5e1d2c60Y1KTPf&source=5176.11533457&userCode=28kqeewo

1）下载PicGo 软件 下载链接：https://github.com/Molunerfinn/PicGo/releases

2）购买阿里云OSS对象存储服务 创建Bucket 权限为公共读

3）点击个人头像，生成keyid和keysecret 使用子用户 创建用户

4）给添加的用户添加权限

5）创建AccessKey

6）设置PicGo

KeyId,KeySecret 为创建的AccessKey

存储空间名为 Bucket的名称

存储区域为 oss-cn-shenzhen (shenzhen服务器地址)

点击确定，然后点击设为默认图床

7）大功告成，打开Typora测试 插入图片自动上传到阿里云OSS对象存储

# Docker部署GitLab

拉取docker镜像

docker pull gitlab/gitlab-ce:10.2.2-ce.0
1

运行镜像

docker run -d -p 443:443 -p 9888:80 -p 222:22 \
--name gitlab --restart always \
-v /opt/module/gitlab/config:/etc/gitlab \
-v /opt/module/gitlab/logs:/var/log/gitlab \
-v /opt/module/gitlab/data:/var/opt/gitlab \
gitlab/gitlab-ce:10.2.2-ce.0


# \ 是换行符  建议去掉合并成一行执行

# -d 代表后台运行
# -p 是指映射端口，其中22用于ssh客户代码，443是https，80是页面的访问端口。
# 注意冒号前边是宿主机的端口，后边是容器内的端口，也就是你需要修改的冒号前边的。

# -v 是目录挂载，config目录里将来我们可以自定义配置。或者我们需要备份数据，就可以可以备份这三个目录。
# 同样，前边是宿主机的。

# --name 是容器名称
# --restart 会在docker启动的时候启动容器

# 最后一行是容器使用的镜像
